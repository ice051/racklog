12 Racklog原语的术语表

12.1 Racket 谓词

procedure

(logic-var? x) → boolean?

  x : any/c
标识一个逻辑变量。

procedure

(atomic-struct? x) → boolean?

  x : any/c
标识 (current-inspector) 无法检查的结构。

procedure

(atom? x) → boolean?

  x : any/c
标识可能出现在Racklog程序中的原子值。相当于合同 (or/c boolean? number? string? bytes? char? symbol? regexp? pregexp? byte-regexp? byte-pregexp? keyword? null? procedure? void? set? atomic-struct?) 。

procedure

(compound-struct? x) → boolean?

  x : any/c
标识 (current-inspector) 可以检查的结构。

procedure

(compound? x) → boolean?

  x : any/c
标识可能出现在Racklog程序中的复合值。相当于合同 (or/c pair? vector? mpair? box? hash? compound-struct?) 。

procedure

(unifiable? x) → boolean?

  x : any/c
标识可能出现在Racklog程序中的值。本质上要么是一个 atom? 、 logic-var? 或者包含 unifiable? 的 compound? 。

procedure

(answer-value? x) → boolean?

  x : any/c
标识可能出现在 answer? 中的值。本质上是 unifiable? 不包含 logic-var? 。

procedure

(answer? x) → boolean?

  x : any/c
标识由 %more 和 %which 返回的答案。相当于合约 (or/c false/c (listof (cons/c symbol? answer-value?))) 。

value

goal/c : contract?

为目标而订立的契约。

12.2 用户界面

syntax

(%which (V ...) G ...)

 
  V : identifier?
  G : goal/c
返回满足 G 的变量 V 的 answer? ，如果 G 不能满足，则返回 #f 。如果可用的话，调用 %more 会产生更多的实例化。

procedure

(%more) → answer?

%more 会以最近的 %which 形式产生更多变量实例，以满足 %which 形式的目标。如果找不到更多的解决方案， %more 将返回 #f 。

syntax

(%find-all (V ...) G ...)

 
  V : identifier?
  G : goal/c
比如 (list (%which (V ...) G ...) (%more) ...) ，加上 (%more) ，就像答案一样多。(如果有无限多的答案，这将不会终止。)

12.3 关系

syntax

(%rel (V ...) clause ...)

 
clause = [(E ...) G ...]
 
  V : identifier?
  E : expression?
  G : goal/c
返回一个谓词函数。每个子句 C 都表示，如果所有目标 G (E 都能够成功，那么通过将谓词对象应用到任何匹配 ...) 的对象所创建的目标就被认为是成功的。变量 V 是 clause 的局部逻辑变量

procedure

(%empty-rel E ...) → goal/c

  E : unifiable?
目标 (%empty-rel E ...) 总是失败。 值 %empty-rel 用作谓词的起始值，稍后可以使用 %assert! 和 %assert-after! 增强谓词。

syntax

(%assert! Pname (V ...) clause ...)

 
  Pname : identifier?
  V : identifier?
将子句 clauses 添加到谓词(Racket 变量 Pname 的值)的 末尾 。变量 V 是 clause 的局部逻辑变量

syntax

(%assert-after! Pname (V ...) clause ...)

 
  Pname : identifier?
  V : identifier?
类似于 %assert! ，但是将新的子句添加到现有谓词的 前面 。

12.4 Racklog 变量

procedure

(_) → logic-var?

产生新逻辑变量的程序。可以用于我们想要一个逻辑变量但不想命名它的情况。(相比之下， %let 为它创建的逻辑变量引入新的词法名称。)

syntax

(%let (V ...) expr ...)

 
  V : identifier?
引入 V 作为 expr 中使用的词法作用域逻辑变量。

12.5 截断

syntax

(%cut-delimiter . any)

引入截断。参见 截断 (!) 。

syntax

!

截断目标，参见 截断 (!) 。

只能在 %cut-delimiter 或 %rel 中语法上使用。

12.6 Racklog 运算符

value

%fail : goal/c

目标 %fail 总是失败。

value

%true : goal/c

目标 %true 成功。重试失败。

procedure

(%repeat) → goal/c

目标 (%repeat) 总是成功的(即使是重试)。对于故障驱动循环很有用。

syntax

(%and G ...)

 
  G : goal/c
目标 (%and G ...) 成功，如果所有目标 G, ..., 成功。

syntax

(%or G ...)

 
  G : goal/c
目标 (%or G ...) 成功，如果其中一个 G, ..., 按照这个顺序尝试，成功。

procedure

(%not G) → goal/c

  G : goal/c
如果 G 失败，目标 (%not G) 成功。

procedure

(%if-then-else G1 G2 G3) → goal/c

  G1 : goal/c
  G2 : goal/c
  G3 : goal/c
目标 (%if-then-else G1 G2 G3) 首先尝试 G1 ；如果成功，尝试G2 ；如果不是，尝试 G3 。

12.7 统一

procedure

(%= E1 E2) → goal/c

  E1 : unifiable?
  E2 : unifiable?
如果 E1 可以与 E2 统一，则目标 (%= E1 E2) 成功。逻辑变量的任何结果绑定都将被保留。

procedure

(%/= E1 E2) → goal/c

  E1 : unifiable?
  E2 : unifiable?
%/= 是 %= 的否定。如果 E1 不能与 E2 统一，则目标 (%/= E1 E2) 成功。

procedure

(%== E1 E2) → goal/c

  E1 : unifiable?
  E2 : unifiable?
如果 E1 与 E2 相同，则目标 (%== E1 E2) 成功。它们在结构上应该是相等的。如果包含逻辑变量，它们应该在相同的位置有相同的变量。与 %= 调用不同，这个目标不会绑定任何逻辑变量。

procedure

(%/== E1 E2) → goal/c

  E1 : unifiable?
  E2 : unifiable?
%/== 是 %== 的否定。如果 E1 和 E2 不相同，则目标 (%/== E1 E2) 成功。

syntax

(%is E1 E2)

目标 (%is E1 E2) 与 E1 统一 E2 作为 Racket 表达式的计算结果。 E2 可能包含自动解引用的逻辑变量。如果 E2 包含未绑定的逻辑变量，则失败。

parameter

(use-occurs-check?) → boolean?

(use-occurs-check? on?) → void?
  on? : boolean?
如果是false(默认值)，Racklog的统一将不会使用occurs检查。如果为真，则启用occurs检查。

12.8 数值谓词

procedure

(%< E1 E2) → goal/c

  E1 : unifiable?
  E2 : unifiable?
如果 E1 和 E2 绑定数字，并且 E1 小于 E2 ，则目标 (%< E1 E2) 成功。

procedure

(%<= E1 E2) → goal/c

  E1 : unifiable?
  E2 : unifiable?
如果 E1 和 E2 与数字绑定，而 E1 小于或等于 E2 ，则目标 (%<= E1 E2) 成功。

procedure

(%=/= E1 E2) → goal/c

  E1 : unifiable?
  E2 : unifiable?
如果 E1 和 E2 绑定数字，并且 E1 不等于 E2 ，则目标 (%=/= E1 E2) 成功。

procedure

(%=:= E1 E2) → goal/c

  E1 : unifiable?
  E2 : unifiable?
如果 E1 和 E2 与数字绑定， E1 与 E2 相等，则目标 (%=:= E1 E2) 成功。

procedure

(%> E1 E2) → goal/c

  E1 : unifiable?
  E2 : unifiable?
如果 E1 和 E2 与数字绑定， E1 大于 E2 ，则目标 (%> E1 E2) 成功。

procedure

(%>= E1 E2) → goal/c

  E1 : unifiable?
  E2 : unifiable?
如果 E1 和 E2 与数字绑定，且 E1 大于或等于 E2 ，则目标 (%>= E1 E2) 成功。

12.9 列表谓词

procedure

(%append E1 E2 E3) → goal/c

  E1 : unifiable?
  E2 : unifiable?
  E3 : unifiable?
如果 E3 与通过添加 E1 和 E2 获得的列表是统一的，则目标 (%append E1 E2 E3) 成功。

procedure

(%member E1 E2) → goal/c

  E1 : unifiable?
  E2 : unifiable?
如果 E1 是 E2 中的列表成员，则目标 (%member E1 E2) 成功。

12.10 集合谓词

procedure

(%set-of E1 G E2) → goal/c

  E1 : unifiable?
  G : goal/c
  E2 : unifiable?
目标 (%set-of E1 G E2) 与 E2 统一目标 G 成功的所有 E1 实例化的 集合 。

procedure

(%set-of-1 E1 G E2) → goal/c

  E1 : unifiable?
  G : goal/c
  E2 : unifiable?
类似于 %set-of，但是如果集合为空则失败。

procedure

(%bag-of E1 G E2) → goal/c

  E1 : unifiable?
  G : goal/c
  E2 : unifiable?
目标 (%bag-of E1 G E2) 将目标 G 成功的所有 E1 实例化的 包 (多集)与 E2 统一。

procedure

(%bag-of-1 E1 G E2) → goal/c

  E1 : unifiable?
  G : goal/c
  E2 : unifiable?
类似于 %bag-of ，但是如果包为空则失败。

syntax

(%free-vars (V ...) G)

 
  V : identifier?
  G : goal/c
识别变量 V ...，在目标 G 中是自由的。它用于在设置谓词 (%bag-of 、 %set-of 等等)的调用中避免存在量化。

12.11 Racklog 谓词

procedure

(%compound E) → goal/c

  E : unifiable?
如果 E 是一个复合值，则目标 (%compound E) 成功。

procedure

(%constant E) → goal/c

  E : unifiable?
如果 E 是一个原子值，则目标 (%constant E) 成功。

procedure

(%var E) → goal/c

  E : unifiable?
如果 E 没有完全实例化，则目标 (%var E) 成功，即它至少有一个未绑定变量。

procedure

(%nonvar E) → goal/c

  E : unifiable?
%nonvar 是 %var 的否定。如果 E 完全实例化，则目标 (%nonvar E) 成功，也就是说，它没有未绑定的变量。

12.12 高阶谓词

procedure

(%apply P L) → goal/c

  P : unifiable?
  L : unifiable?
如果 L 是一个包含元素 E... 的列表，则目标 (%apply P L) 成功。并且如果 P 是一个接受 E 一样多参数的谓词，并且如果目标 (P E ...) 成功。

如果 L 没有足够的实例化来确定它的长度，那么目标就会失败。

例如，目标

(%apply %= (list 1 X))

相当于

(%= 1 X)

如果 X 可以与 1 统一，那么它就成功。

procedure

(%andmap P L ...+) → goal/c

  P : unifiable?
  L : unifiable?
如果目标 (%andmap P L ...) 成功，则所有值 L 都是等长的列表，如果目标 (P E ...) 成功，则每组值 E 依次从每个列表 L 获取。

作为一个例子，特别是目标 (%andmap %<= '(1 2 3) '(4 5 6)) 等同于
(%and (%<= 1 4)
      (%<= 2 5)
      (%<= 3 6))

12.13 Racklog 操纵变量

procedure

(%freeze S F) → goal/c

  S : unifiable?
  F : unifiable?
目标 (%freeze S F) 与 F 结合，是 S 中结构的一个新的冻结版本。冻结意味着保留所有未绑定的变量。因此， F 可以被用作 绑定 对象，而不用担心它的变量被统一绑定。

procedure

(%melt F S) → goal/c

  F : unifiable?
  S : unifiable?
目标 (%melt F S) 将 S 与 F 中冻结结构的解冻(原始)形式统一起来。

procedure

(%melt-new F S) → goal/c

  F : unifiable?
  S : unifiable?
目标 (%melt-new F S) 将 S 与 F 中冻结结构的解冻 副本 统一起来。这意味着 F 中的未绑定逻辑变量使用新的逻辑变量。

procedure

(%copy F S) → goal/c

  F : unifiable?
  S : unifiable?
目标 (%copy F S) 与 F 中冻结结构的复制 S 统一。
