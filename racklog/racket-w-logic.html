3 在 Racklog 使用传统 Racket 表达方式

Racklog 谓词的参数可以是任何 Racket 对象。特别是，可以使用列表、向量、字符串、散列表等复合结构，以及使用完整的Racket构造函数和分解运算符数组的Racket表达式。例如，考虑以下目标：

(%member x '(1 2 3))

这里， %member 是一个谓词， x 是一个逻辑变量， '(1 2 3) 是一个结构。给 %member 一个适当直观的定义，对于 x = 1 、 2 和 3 ，上述目标成功。

现在定义谓词，比如 %member ：

(define %member
  (%rel (x y xs)
    [(x (cons x xs))]
    [(x (cons y xs))
      (%member x xs)]))

例如， %member 定义三个局部变量：x 、 y 和 xs 。它有两个子句，标识确定成员资格的两种方式。

%member 的第一个子句陈述一个事实：对于任意 x ， x 是列表成员， x 也是该列表的头部。

%member 的第二个子句是一个规则：如果能证明 x 是列表 尾部 的成员，那么它就是列表的成员。换句话说，原始的 %member 目标被转换成一个 子 目标，这也是一个 %member 目标。

注意 %member 定义中的变量 y 在第二个子句中只出现一次。因此，不需要为其命名。(名称只能帮助将第二个匹配项与第一个匹配项匹配。)Racklog允许使用表达式 (_) 来表示匿名变量。(也就是说， _ 是在每次调用时生成一个新的匿名变量的工具。)谓词 %member 可以重写为

(define %member
  (%rel (x xs)
    [(x (cons x (_)))]
    [(x (cons (_) xs))
      (%member x xs)]))

3.1 构造函数

可以使用构造器 — 用于创建结构的 Racket 程序 — 来模拟Racklog中的数据类型。例如，定义一个自然数字数据类型，其中 0 表示零， (succ x) 表示前身为 x 的自然数。构造函数 succ 在Racket中定义为：

(define succ
  (lambda (x)
    (vector 'succ x)))

加法和乘法可以定义为:

(define %add
  (%rel (x y z)
    [(0 y y)]
    [((succ x) y (succ z))
      (%add x y z)]))
 
(define %times
  (%rel (x y z z1)
    [(0 y 0)]
    [((succ x) y z)
     (%times x y z1)
     (%add y z1 z)]))

可以用它做很多运算，例如，阶乘谓词看起来像：

(define %factorial
  (%rel (x y y1)
    [(0 (succ 0))]
    [((succ x) y)
      (%factorial x y1)
      (%times (succ x) y1 y)]))

3.2 %is

上面是进行算术的一种非常低效的方法，尤其是当基础语言Racket提供出色的算术功能（包括全面的“塔”和精确的有理算术）时。 直接在Racklog子句中使用Racket计算的一个问题是，使用的表达式可能包含需要取消引用的逻辑变量。Racklog提供处理这个问题的谓词 %is 。我们的目标

(%is X E)

将 X 与 E 的值统一起来作为 Racket 表达式。 E 可以有逻辑变量，但通常它们至少应该是有约束的，因为没有约束的变量对于 E 中使用的不法操作者来说可能不是合适的值。

现在可以直接使用 Racket 算术来写一个更有效的 %factorial 谓词：

(define %factorial
  (%rel (x y x1 y1)
    [(0 1)]
    [(x y) (%is x1 (- x 1))
           (%factorial x1 y1)
           (%is y (* y1 x))]))

这种效率带来的代价是，我们只能在第一个参数已经实例化的情况下使用 %factorial 。在许多情况下，这是合理的限制。事实上，考虑到这个限制，没有什么可以阻止我们直接使用 Racket 的阶乘：

(define %factorial
  (%rel (x y)
    [(x y)
     (%is y (racket-factorial
              x))]))

或者更好的是，“内联”任何调用 %factorial 与 %is 表达式调用的 Racket 阶乘 ，后者的定义方式通常是:

(define racket-factorial
  (lambda (n)
    (if (= n 0) 1
        (* n (racket-factorial
               (- n 1))))))

3.3 词法域

我们可以使用Racket的词法作用域来增强谓词定义。下面是使用隐藏的辅助谓词定义的列表反转谓词：

(define %reverse
  (letrec
    ([revaux
       (%rel (x y z w)
         [('() y y)]
         [((cons x y) z w)
           (revaux y
             (cons x z) w)])])
    (%rel (x y)
      [(x y) (revaux x '() y)])))

(revaux X Y Z) 使用 Y 作为一个累加器，将 X 反向到 Z 。(Y 开始是 () 。 X 的每个头都与 Y cons 。最后，当 X 缩减到 () 时， Y 包含反向列表，可以作为 Z 返回。)

revaux 纯粹用作 %reverse 的辅助谓词，因此它可以隐藏在词法轮廓中。用 letrec 代替 let ，因为 revaux 是一个递归过程。

3.4 类型谓词

Racklog 提供两个谓词，让用户可以检测对象的类型。

目标

(%constant X)

如果 X 是一个 原子 对象，则成功。

谓词 %compound ，即 %constant 的否定形式，检查它的参数是否不是原子对象。

以上只是相应的 Racket 谓词的逻辑编程等价。在 Racklog，用户可以使用谓词 %is 和 Racket 谓词来写更多的类型检查。因此，要测试 X 是否是一个字符串，可以使用以下目标：

(%is #t (string? X))

除了原始的 Racket 谓词外，用户定义的 Racket 谓词也可以这样导入。
